Datapeer provides official SDKs so you can integrate event tracking directly into your favorite language or framework.

This page gives an overview of which features are currently supported across our SDKs, including event capture, user identification, and autocapture.

## SDK Feature Support

| SDK                                | Event Capture | User Identification | Autocapture |
| ---------------------------------- | :-----------: | :-----------------: | :---------: |
| [JavaScript (Web)](/untitled-page) | ✅             | ✅                   | ✅           |

## Feature Definitions

- **Event Capture**\
  Send custom events such as `page_view`, `signup`, or `purchase` to Datapeer.
- **User Identification**\
  Associate events with a unique user using a `distinct_id`, enabling user-level analytics, session tracking, and funnels.
- **Autocapture**\
  Automatically track common interactions (such as clicks and page views) without requiring manual instrumentation.

## Legend

- ✅ Fully supported
- ⚠️ Supported with limitations
- ❌ Not supported
